Projeto: Sistema Inteligente de Apoio a Diagnósticos Médicos por Imagem (SIAD-MI)

2. Backend (Java + Spring Boot)

* Frameworks:

  * `Spring Boot` – para criação de APIs RESTful
  * `Spring Data JPA` – integração com banco de dados

* Funções:

  * Receber requisição via API (imagem + metadados)
  * Enviar imagem ao microserviço Python de IA
  * Armazenar e recuperar dados médicos (paciente, resultado, imagem)

#3.Microserviço Python (para IA)

* Motivo: A IA continua em Python (PyTorch, TensorFlow), comunicando-se com o backend via API REST ou HTTP.
* Tecnologias:

  * `FastAPI` ou `Flask`
  * Servidor separado para a IA (chamado pelo Spring Boot)

4.Banco de Dados ==============================================================================Not Implemented

* H2 Database - Runtime
* JPA/Hibernate para persistência

Fluxo de Funcionamento

plaintext
[Interface web simples]
    ↓ (upload imagem)
[API Java Spring Boot Backend]
    ↓ (chama microserviço de IA Python via REST)
[Serviço de IA Python (FastAPI)]
    ↓ (inferencia com modelo)
    ↑ retorna diagnóstico + probabilidade
[Spring Boot salva no banco e envia para App]
    ↑ (resposta com dados ao App Desktop)

Segurança ==============================================================================Not Implemented

* Comunicação segura via HTTPS

Vantagens dessa Arquitetura

* Separação clara de responsabilidades
* Backend escalável (Spring Boot é robusto)
* IA independente (permite upgrades, reuso e escalabilidade)
